System Architecture & How It Works

🔄 How the Self-Healing System Works

  1. Continuous Monitoring: Simulated sensors (voltage, current, power) track all nodes every 100ms
  2. Fault Detection: AI algorithms detect anomalies using threshold-based & pattern recognition
  3. Fault Isolation: Faulty node is immediately disconnected to prevent cascading failures
  4. Load Redistribution: Power is automatically rerouted through healthy paths
  5. Priority Handling: Critical loads (hospitals) get priority over flexible loads
  6. Self-Recovery: Once fault is resolved, node reconnects and reintegrates into the grid

🧠 Fault Detection Algorithms

1. Threshold Detection: If V < 380V or V > 420V → Voltage Fault
2. Rate of Change: dV/dt > 20V/s → Rapid Fault
3. Power Balance: |Gen - Load| > Threshold → Imbalance
4. Harmonic Analysis: THD > 5% → Distortion
5. Pattern Matching: Historical patterns + ML classification

📊 Load Redistribution Logic

Priority Queue:

1. Critical (Hospitals, Emergency) - 100% supply

2. Important (Office, Infrastructure) - 80% supply

3. Flexible (Factory, Residential) - 50% supply

Energy Source Priority:

1. Local renewable (Solar + Wind)

2. Battery storage (if SoC > 20%)

3. Grid connection (fallback)

⚙️ System Components

Data Simulator

Generates realistic sensor data for voltage, current, power, frequency with weather effects

Fault Detector

Analyzes real-time data using ML/AI to identify faults with <100ms latency

Recovery Engine

Automatically isolates faults and reroutes power maintaining system stability

Digital Twin

Simulates grid scenarios offline to test recovery strategies before deployment

✅ Advantages Over Traditional Grids

  • ✓ Self-Healing: Automatic fault isolation without human intervention
  • ✓ Resilience: Continues operation even with node failures (islanding capability)
  • ✓ Cost Reduction: 20-40% lower operational costs by optimizing renewable usage
  • ✓ Faster Response: Sub-100ms fault response vs. seconds in traditional grids
  • ✓ Sustainability: Prioritizes renewable energy with battery buffering
  • ✓ Smart Priority: Guarantees power to critical loads during stress

⚠️ Limitations of Software-Only Implementation

  • ⚠️ No Physical Sensors: Cannot detect actual hardware failures - relies on simulated data
  • ⚠️ Communication Delay: Real systems have network latency; simulation assumes instant
  • ⚠️ Protection Equipment: Requires physical relays/switches to actually isolate faults
  • ⚠️ Accuracy: Model accuracy depends on quality of simulated data generation
  • ⚠️ Edge Cases: Rare scenarios (cascading failures) difficult to validate
  • ⚠️ Cyber Security: Software vulnerabilities could compromise the entire system
